# 노인 안부 상담 AI — Claude Code 구현 지시서

## 0. 작업 방식 (가장 중요)
- 이 문서만으로 **끝까지 자율 진행**한다. 단계 사이에 확인을 구하거나 멈추지 말 것.
- 기술 선택지가 갈리면 **질문 대신 스스로 결정**하고 진행한다. 아래 스택은 기본안일 뿐, 더 나은 판단이 있으면 알아서 바꿔라.
- 유일하게 멈춰도 되는 경우: **API 키가 없어 실제 호출 검증이 불가능할 때.** 그 경우에도 멈추지 말고 → `MOCK_MODE=true`(가짜 STT/TTS/LLM/OCR 응답)로 전체 플로우를 먼저 완성한 뒤, `.env`에 어떤 키를 채워야 하는지와 NCP 콘솔 발급 경로를 README에 정리해두고 계속 진행하라. 키가 채워지면 실제 연동을 검증한다.
- 모델·API는 **사용 가능한 것 중 가장 좋은 것**을 써라. 비용은 지원 계정이라 신경 쓰지 않는다. (예: HyperCLOVA X 최상위 모델, 프리미엄 TTS 보이스)
- 각 단계 완료 시 서버를 직접 띄워 자체 테스트하고, git commit 후 다음 단계로 넘어가라.
- 각 CLOVA API의 엔드포인트·인증·요청/응답 형식·모델명은 **공식 NCP 문서 기준 최신 사양**으로 구현하라. 기억으로 추측하지 말고, 문서 확인이 필요하면 나에게 해당 페이지를 요청해라 (단, 그동안 mock으로 진행은 계속).

## 1. 프로젝트 개요
독거노인 대상 **AI 안부 상담**을 웹에서 제공하는 데모 서비스.
- 사용자는 어르신(또는 데모 시연자). AI가 상담원 역할로 **먼저 안부 인사를 건네고**, 채팅하듯 자연스럽게 티키타카 대화한다.
- 대화 내용이 **실시간으로 화면에 기록**되고, AI가 대화 속 **특이사항(건강·정서·인지·사기 노출·복지 니즈·긴급)을 자동 추출해 옆 패널에 실시간 정리**한다.
- 상황에 맞게 **자격 맞는 복지 정보 안내**와 **사기 예방 코칭**을 대화에 자연스럽게 녹인다.
- **서류 사진을 채팅에 첨부**하면 OCR로 내용을 추출해 쉬운 말로 설명하고, 잘못 기재된 부분·주의사항·사기 의심 여부를 알려준다.
- 세션 종료 시 **전체 요약 리포트**를 보여준다.

**한 줄 컨셉:** 한국 복지는 '신청주의'라 알아야 받는다. AI가 먼저 안부를 물으며 자격 맞는 복지를 알려주고 사기 예방까지 도와, 노인의 정보 비대칭과 고립을 메운다.

## 2. UX 상세
채팅 중심 인터페이스 (모바일에서도 보기 좋게):
- **메인 — 채팅창**: 세션 시작 시 AI가 먼저 인사. 텍스트 입력 기본, 말풍선에 화자·시간 표시.
- **마이크 버튼**: 누르고 말하면(푸시투토크) 녹음 → CLOVA Speech STT → 사용자 말풍선으로 등록 → AI 응답.
- **AI 응답 음성**: CLOVA Voice TTS로 자동 재생 (상단에 음성 on/off 토글). 노인 친화적으로 차분하고 또렷한 프리미엄 보이스 선택.
- **파일 첨부 버튼**: 이미지(고지서·안내문·신청서·의심 문자 캡처) 업로드 → CLOVA OCR → 추출 내용을 바탕으로 AI가 채팅으로 설명 + 수정/주의/사기 판별.
- **사이드 패널 (데스크톱은 우측, 모바일은 탭 전환)**:
  - 특이사항 카드: 카테고리·심각도(초록/노랑/빨강) 색상, 실시간 추가·갱신
  - 매칭된 복지 정보: 대화에서 자격 신호가 잡힌 복지 항목 목록
- **세션 종료 버튼** → 요약 리포트 모달 (특이사항 정리 + 후속 권고 + 안내한 복지 목록).
- 글자 크게, 대비 높게, 버튼 큼직하게 — 노인 사용자 가정한 접근성.

## 3. 멀티유저 (동시 접속)
- 접속자마다 **완전히 독립된 세션**: 대화 이력·특이사항·복지 매칭·리포트가 세션 단위로 격리. 공유 상태 없음.
- 접속 시 세션 ID 발급, 세션별 WebSocket 연결. 동시 접속 수 제한 없이 서로 간섭 없어야 함.
- 앱은 상태를 세션 객체에만 두어 필요 시 인스턴스를 여러 개 띄워 수평 확장 가능한 구조로. (기본 데모는 단일 인스턴스로 충분)

## 4. 기술 스택 (기본안 — 스스로 개선 가능)
- **백엔드**: Python + FastAPI, 실시간 갱신은 WebSocket.
- **프론트**: 단일 페이지 HTML + 순수 JS + Tailwind(CDN). 빌드 도구 없이. 프론트도 전부 네가 작성 (나는 웹 프론트 경험 적음).
- **LLM**: NCP CLOVA Studio **HyperCLOVA X 최상위 모델** (콘솔 제공 목록 확인, 예: HCX-007 계열). 대화 응답과 특이사항 추출은 **분리된 호출**로: 응답은 즉시 스트리밍/반환, 추출은 비동기로 패널만 갱신. 추출은 structured/JSON 출력 사용. ※ CLOVA Studio는 추론(thinking)·Structured Output·이미지 입력 등 일부 기능 동시 사용 제약이 있으니 문서 확인 후 모델/옵션 조합을 정할 것. 지연이 문제면 응답용·분석용 모델을 다르게 써도 됨.
- **STT**: CLOVA Speech (턴 단위 인식이면 충분, 스트리밍 인식은 스트레치).
- **TTS**: CLOVA Voice 프리미엄 보이스.
- **OCR**: CLOVA OCR (문서 성격에 맞는 General/Document 계열 선택).
- **복지 지식 grounding**: `knowledge/welfare.json` — 기초연금, 긴급복지지원, 의료급여, 에너지바우처, 노인맞춤돌봄, 주거급여 등 **주요 복지 8~12개**를 실제 기준(대상·조건·금액·신청처)으로 정리해 두고, 복지 안내 시 이 파일 내용에 근거해 답하게 한다. 임베딩 기반 RAG는 스트레치.
- **키 관리**: `.env`(python-dotenv) + `.env.example`. 하드코딩·커밋 금지. `.gitignore` 포함.
- **문서화**: README에 실행법, .env 항목별 설명, NCP 콘솔 키 발급 경로, 아키텍처 개요.

## 5. 특이사항 추출 스키마
매 턴, 누적 대화(+OCR 결과)를 분석해 아래 형식의 목록을 갱신:
```json
{
  "카테고리": "건강 | 정서 | 인지 | 사기_노출 | 복지_니즈 | 긴급",
  "내용": "관찰된 사실 요약 (단정·진단 아님)",
  "심각도": "낮음 | 보통 | 높음",
  "사람_개입_필요": true
}
```
- 건강: 수면·식사·통증·복약·낙상 등 언급 / 정서: 외로움·우울감·불안 표현 / 인지: 반복·혼란 신호(스크리닝 참고) / 사기_노출: 의심 전화·문자, 금전·개인정보 요구 / 복지_니즈: 경제적 어려움·독거·건강 부담 등 자격 신호 / 긴급: 자·타해, 급성 응급 → 즉시 사람 개입.
- 심각도 '높음' 또는 긴급이면 UI 강조 + "보호자/담당자 연결 권고" 표시.

## 6. 가드레일 (필수 — 어떤 리팩토링에서도 유지)
1. **진단 금지.** 건강·정서·인지 항목은 전부 '관찰'로만. "우울증/치매입니다" 같은 단정 절대 금지. 가족·사회복지사·의료진 참고용으로 표현.
2. **긴급은 사람에게.** 자·타해·응급 신호는 AI가 해결하려 하지 말고 "즉시 보호자/담당자/119 연결 권고" 플래그만. 실제 연락처를 지어내지 말 것.
3. **복지 정보 정확성.** 액수·자격은 `welfare.json` 근거로만. 파일에 없으면 "정확한 내용은 복지로(129)·주민센터 확인" 안내. 지어내지 말 것.
4. **프라이버시.** 데모는 가상 데이터 전제. 업로드 이미지는 세션 메모리 내 처리 위주, 디스크 저장 최소화하고 세션 종료 시 삭제. 대화 이력도 영구 저장하지 않음. (국내 처리 '소버린 AI'는 발표 강점으로 README에 한 줄)
5. **위로하되 치료 아님.** 힘든 감정 표현엔 따뜻하게 공감하되 심리치료 시도 금지, 필요 시 사람 연결 플래그.
6. **말투.** 어르신 대상: 존댓말, 짧은 문장, 쉬운 단어, 한 번에 하나씩.

## 7. 구현 순서 (자율 진행, 단계별 commit)
- **0단계**: 프로젝트 골격 + FastAPI 서버 + 세션 관리 + `.env`/`.env.example`/`.gitignore` + MOCK_MODE 스위치.
- **1단계**: 채팅 루프 — 텍스트 입출력 + HyperCLOVA X(또는 mock) 연동, AI 선인사 포함.
- **2단계**: 특이사항 추출 + 사이드 패널 실시간 갱신(WebSocket).
- **3단계**: OCR — 이미지 첨부 → 추출 → 설명/수정/사기 판별 응답 + 특이사항 반영.
- **4단계**: STT — 마이크 녹음 → 텍스트 → 채팅.
- **5단계**: TTS — AI 응답 자동 음성 재생 + 토글.
- **6단계**: 복지 grounding(`welfare.json`) + 복지 패널 + 종료 요약 리포트.
- **7단계**: UI 다듬기(접근성·모바일) + README 완성 + 데모 시나리오 스크립트 1개 작성(발표용).
- **스트레치**: 스트리밍 STT, 임베딩 RAG, Papago 다국어, 대화 중 인지 스크리닝 리포트 고도화.

## 8. 완료 기준
- `MOCK_MODE=true`로 키 없이도 전체 UX가 동작.
- 키 입력 시 STT·TTS·LLM·OCR 실제 연동 동작.
- 동시 접속 2개 브라우저에서 세션이 섞이지 않음.
- 가드레일 6개 항목이 시스템 프롬프트/코드에 실제 반영되어 있음.
- README만 보고 제3자가 실행 가능.
